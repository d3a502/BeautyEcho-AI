# 系统架构说明

## 总体架构

妆声引擎 AI 采用“数据底座 + 飞书 AI + 多 Agent + 策略闭环”的架构。

```text
数据层
  社媒样本 / 用户评论 / 竞品案例 / 产品 brief / 投放反馈

协作与 AI 底座
  飞书多维表格 / 飞书 AI / 飞书文档

分析 Agent 层
  妆声雷达 / 趋势声纹 / 竞品回响 / 用户共鸣 / 产品映射

策略 Agent 层
  灵感工坊 / 合规校音 / 复盘回路

输出层
  内容策略报告 / 达人 brief / 小红书笔记方向 / 抖音脚本 / 复盘总结
```

## Agent 分工

| Agent | 输入 | 输出 |
|---|---|---|
| 妆声雷达 | 社媒标题、正文、互动数据 | 平台热词、热门内容形式 |
| 趋势声纹 | 热门内容和用户表达 | 审美趋势、场景偏好 |
| 竞品回响 | 竞品内容和评论 | 竞品机会点、差异化打法 |
| 用户共鸣 | 用户评论、测评反馈 | 痛点聚类、购买顾虑 |
| 产品映射 | 产品 brief、用户需求 | 主推人群、主卖点、避雷表达 |
| 灵感工坊 | 策略 brief | 多平台内容策略和资产 |
| 合规校音 | 内容草稿、产品事实 | 风险提示、修改建议 |
| 复盘回路 | 投放结果、评论反馈 | 复盘结论、策略库更新 |

## 数据闭环

1. 社媒样本进入飞书多维表格。
2. 飞书 AI 做摘要、标签和初筛。
3. 分析 Agent 输出趋势、竞品和用户洞察。
4. 产品映射 Agent 对齐产品事实和用户意愿。
5. 灵感工坊生成内容策略。
6. 合规校音检查风险表达。
7. 投放反馈进入复盘回路。
8. 结果沉淀为下一轮策略库。
